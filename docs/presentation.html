Claude Code: AI that lives in your project

What it is

  • A command-line AI assistant
  • Reads your entire project — thesis, code, data
  • Follows custom instructions you write
  • Executes tools: reads files, runs code, searches

What it is NOT

  • It doesn't know your field by default
  • It can't invent papers or data
  • It needs your thinking — it amplifies it
You run it with: claude .
The key insight: Claude Code sees your whole project, not just a chat window. It reads CLAUDE.md for your custom rules, then acts on your codebase.

LLMs vs AI Agents: what's the difference?

LLM

Large Language Model

  • Predicts the next token given a prompt
  • Stateless — no memory between calls
  • No tools, no actions, text in → text out
  • Examples: GPT-4, Claude, Gemini (the model itself)
→

AI Agent

LLM + tools + memory + planning

  • Perceives — reads your files, runs searches
  • Plans — breaks a task into steps
  • Acts — edits files, runs code, calls APIs
  • Remembers — reads your CLAUDE.md every session
Claude Code is an agent. The LLM is its brain — the agent is what acts on your project.
The distinction matters: students who think of Claude as a chatbot will underuse it. It can read every file in the project, run Python, compile their PDF — not just answer questions.

Skills: instructions the agent follows

A skill is a Markdown file stored at
.claude/skills/<name>/SKILL.md

When you type /write-section, Claude Code loads that file and follows the instructions inside — step by step, every time.

Think of it as a recipe: you provide the ingredients (your notes), the skill provides the method.

You can write your own skills.
Any .claude/skills/yourskill/SKILL.md becomes /yourskill.

---
name: write-section
description: Draft a thesis section
  from the student's notes.
---

## Process

1. Read docs/student-profile.md
2. Take the student's bullet notes
3. Write in academic English
4. Use cautious language:
   "suggests", "consistent with"
5. Never add citations not in notes
6. Output clean Pandoc Markdown

Actual excerpt from .claude/skills/write-section/SKILL.md

Skills are just text files — students can read them, copy them, and write their own. This demystifies AI tooling: it's not magic, it's structured instructions.

For quantitative theses: two key skills

Design your strategy

/empirical-design

  • Defines your dependent + independent variables
  • Writes your baseline regression in LaTeX
  • Flags identification concerns
  • Suggests robustness checks
  • Produces expected tables and figures list
  • Gives an interpretation guide
Run before you touch any data

Understand your data

/data-explorer

  • Schema: column names, types, missing value counts
  • Data quality flags: duplicates, outliers, implausible values
  • Suggested summary statistics for thesis section
  • Python code stub for your analysis script
  • Connection to your research question
/data-explorer data/clean/firm_panel.csv
Run when data/clean/ has your file
These two skills are the entry points for any quantitative chapter. Empirical-design runs before data work; data-explorer runs once the clean dataset exists.

Agents: specialists you can call in

Skills = workflows you trigger.  |  Agents = specialist reviewers Claude can delegate to.

literature-reviewer

Checks your lit review isn't a list of summaries. Finds missing streams. Flags overclaiming and unsupported assertions.

methods-reviewer

Reviews your econometric strategy. Flags weak identification, missing controls, and standard error choices that won't pass a seminar.

thesis-editor

Edits any section for clarity and academic tone. Preserves your voice. Never adds facts. Never removes your argument.

Call them naturally: "Review my literature review" or "Edit section 5 for clarity"
Agents are always-available specialists. Students don't need to think about when to use them — just describe what you want and Claude delegates automatically.

Hooks: rules that run automatically

🛡️
protect-raw-data
Fires before any file edit. If the target is data/raw/, it blocks the operation. Your raw data is physically untouchable — no matter what you type.
📋
log-action
After every tool use, appends a timestamped line to .claude/logs/activity.log. Full audit trail of everything Claude did during your session.
🔔
notify
When a long-running task (Pandoc compile, reference validation) finishes, sends a desktop notification. You can go make coffee.
You never configure these. They just work.

✓ Confirmed via Semantic Scholar: 14 references
⚠ Unverified (check manually): 2 references
✗ Key in text but missing from .bib: 1 reference
Never cite a paper you haven't read. /validate-references catches errors, not ethics.
The validate-references skill checks every citation key against the .bib file and then attempts verification via Semantic Scholar. It reports but does not fix — the student must resolve each issue.

Getting ready to defend

  • Flags specific overclaiming sentences
  • Gives you time to prepare answers before the real thing
"Your identification strategy relies on the assumption that [X]. What evidence do you have that this holds in your sample?"

You still practice. But you start from something real.
These two skills are designed to be run together in the final weeks. The adversarial reviewer finds the gaps; the defense slides give the student a starting deck to iterate on.

The contract

AI CAN

  • Draft sections from your notes
  • Suggest search queries — never invent papers
  • Check your methodology for common errors
  • Compile your PDF automatically
  • Tell you hard questions your examiner will ask

AI CANNOT

  • Invent data or empirical results
  • Write your thesis without your thinking
  • Replace your judgment on your topic
  • Know what your professor wants unless you tell it
This slide is the most important one. Print it. Put it above your desk. The moment students try to make AI think for them, the output collapses in quality.

AI + integrity: the line is clear

🔵
Your thinking is the thesis
Claude drafts from your notes. If the notes are empty, the draft is empty. The ideas must be yours.
🔵
Every citation must be real
Claude will never invent a paper. /validate-references checks against Semantic Scholar. If a citation isn't in your .bib, it won't compile. You are still responsible for reading what you cite.
🔵
Disclose how you used AI
Many programs now require an AI use statement. Ask your supervisor. The log at .claude/logs/activity.log is your audit trail.
Using AI to write faster is fine. Using AI to think for you is not.
This slide addresses the question every student is thinking but not asking. Be direct: the audit log exists, disclosure requirements are growing, and there is no AI shortcut that replaces understanding your own research.
